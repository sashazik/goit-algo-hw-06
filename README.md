# Домашнє завдання: Графи та Алгоритми Пошуку

## Опис моделі
У цьому завданні було побудовано граф, що моделює **транспортну мережу міста**.

* **Вершини:** Ключові локації (наприклад: Центр, Аеропорт, Вокзал).
* **Ребра:** Дороги або маршрути між локаціями.
* **Вага ребер:** Відстань або час у дорозі між вершинами.

---

## Завдання 2: Порівняння алгоритмів DFS та BFS

Ми виконували пошук шляху від вершини **"Аеропорт"** до вершини **"Університет"**.

### 1. DFS (Пошук у глибину)
* **Знайдений шлях:** `['Аеропорт', 'Схід', 'Центр', 'Північ', 'Вокзал', 'Університет']`
* **Логіка:** Алгоритм пішов по першому доступному ребру ("Схід"), потім знову обрав перше ліпше ("Центр") і заглиблювався, поки не досяг цілі. 
* **Особливість:** Він не шукає найкоротший шлях, а намагається пройти якомога далі вглиб графа. Тому шлях вийшов довгим і "звивистим".

### 2. BFS (Пошук у ширину)
* **Знайдений шлях:** `['Аеропорт', 'Промзона', 'Північ', 'Вокзал', 'Університет']`
* **Логіка:** Алгоритм перевіряє всіх сусідів пошарово: спочатку на відстані 1 кроку, потім на відстані 2 кроків і т.д.
* **Особливість:** У незваженому графі BFS гарантовано знаходить шлях із **найменшою кількістю вершин** (мінімум пересадок).

### Висновок щодо різниці
* **BFS** знайшов оптимальний шлях за кількістю пройдених вершин.
* **DFS** знайшов просто один із можливих шляхів, який не є оптимальним.

## Завдання 3: Алгоритм Дейкстри

Для розрахунку реального найкоротшого маршруту ми додали **ваги** (відстань/час) до ребер. Просто рахувати кількість вершин (як у BFS) недостатньо, адже пряма дорога може бути дуже довгою.

* **Результат для маршруту Аеропорт -> Університет:**
  `['Аеропорт', 'Промзона', 'Північ', 'Вокзал', 'Університет']`
* **Сумарна вага (довжина шляху):** `17` (6 + 7 + 3 + 2).

**Примітка:** У даній конфігурації графа шлях Дейкстри збігся з BFS, але математично алгоритм Дейкстри гарантує саме мінімальну суму ваг, а не кількість зупинок.

---

## Загальний висновок

| Алгоритм | Коли використовувати |
|----------|----------------------|
| **DFS** (Глибина) | Для перевірки зв'язності графа або коли потрібно знайти *хоч якийсь* шлях, не обов'язково короткий. |
| **BFS** (Ширина) | Для пошуку шляху з мінімальною кількістю кроків (вершин) у незважених графах. |
| **Dijkstra** (Дейкстра) | Для пошуку найкоротшого (найдешевшого/найшвидшого) шляху у графах з вагами. |
